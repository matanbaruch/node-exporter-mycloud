node_exporter for WD My Cloud EX2 Ultra

A web-installable Prometheus node_exporter app for the Western Digital My Cloud EX2 Ultra (OS 5).

⬇ Download the latest .bin View on GitHub

What it does

Runs Prometheus node_exporter on your NAS, installed the normal WD way — upload a .bin from Apps → Install Manually — then managed from a page right inside the dashboard: set the listen port, flip collector toggles, start/stop the service. The exporter publishes the NAS's hardware and OS metrics over HTTP for a Prometheus server to scrape. Your settings are stored on the persistent data volume, so they survive reboots.

Install

  1. Download MyCloudEX2Ultra_node_exporter_<version>.bin from the latest release.
  2. My Cloud dashboard → Apps → Install an app manually → pick the .bin (see WD's OS5 guide).
  3. Open Apps → Node Exporter, set the listen port (default 9100), pick collectors, click Save & restart.
  4. Scrape http://<nas-ip>:9100/metrics from your Prometheus server.

Build from source

Requires Docker (the WD packaging tool is a Linux binary run inside the build image).

git clone https://github.com/matanbaruch/node-exporter-mycloud
cd node-exporter-mycloud
./build.sh
# -> packages/MyCloudEX2Ultra_node_exporter_<version>.bin

The bundled node_exporter version is auto-bumped daily via a scheduled PR; merging it cuts a new release.

Status & security

Targets the My Cloud EX2 Ultra (firmware 5.33.102, ARMv7). Every change is round-trip-verified in CI (APKG header, armv7 node_exporter, expected files), but this build has not been confirmed on physical hardware yet — on-device execution is your final test.

The metrics endpoint is unauthenticated and binds all interfaces. Anyone who can reach the port can read system metrics. Keep the NAS off the public internet and firewall the port to your Prometheus host — or expose it only over a private overlay (e.g. pair it with tailscale-mycloud and scrape over the tailnet).
